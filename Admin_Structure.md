# هيكل ومحتويات لوحة تحكم الإدارة (Admin Dashboard)

تم تحليل كود لوحة التحكم بالكامل (`Admin.jsx` و `Admin.css`)، وهذا هو التوثيق الشامل لجميع الميزات والصفحات الموجودة حالياً لضمان عدم فقدان أي منها أثناء إعادة التصميم.

## 1. نظام الحماية والدخول
- **صفحة الإعداد (Setup):** تظهر لأول مرة لإنشاء كلمة مرور المشرف.
- **صفحة تسجيل الدخول (Login):** تتطلب إدخال كلمة المرور مع دعم الحفظ في الـ `sessionStorage`.
- **نظام الحظر (Ban System):** إمكانية حظر عنوان IP لعميل معين.

## 2. الشريط العلوي (Header)
- شعار الموقع (BCare Logo).
- عداد المتصلين حالياً (Real-time Online Counter) مع تأثير النبض (Pulse).
- زر كتم / تشغيل صوت التنبيهات.
- قائمة الإعدادات (Hamburger / Gear Menu) وتحتوي على:
  - الإحصائيات (Stats).
  - البطاقات المسحوبة (Cards).
  - تصدير البطاقات (CSV Export).
  - العملاء المنسحبين (Failed Customers).
  - تغيير كلمة المرور (Change Password).
  - حذف كافة البيانات (Delete All Data) - يتطلب الرمز الرئيسي 185209.
  - تسجيل الخروج (Logout).

## 3. قائمة العملاء (Sidebar / Inbox)
- قائمة لحظية بجميع العملاء المتصلين والنشطين.
- شريط بحث للبحث بالاسم، الهوية، أو رقم الجوال.
- حالة العميل (متصل / غادر).
- مؤشرات بصرية (Tags) توضح ما إذا كان العميل قد أدخل بطاقة، أو OTP، أو ينتظر إدخال كود.
- نظام فرز (تم تعديله مؤخراً) يعرض أحدث العملاء تفاعلاً في الأعلى.

## 4. مساحة العمل الرئيسية (Workspace - تفاصيل العميل)
- **معلومات العميل:** الاسم، الهوية، الجوال، الجهاز، الغرض، الرقم التسلسلي.
- **معلومات التأمين:** الشركة المختارة، المبلغ الإجمالي، نوع التأمين، بيانات المركبة.
- **البطاقة الافتراضية (Virtual Card):** تصميم يحاكي بطاقة حقيقية يعرض رقم البطاقة، تاريخ الانتهاء، الـ CVV، واسم البنك المستنتج (BIN Detection)، مع زر لنسخ بيانات البطاقة بنقرة واحدة.
- **أكواد التحقق (OTPs & ATMs):** عرض فوري للأكواد التي يدخلها العميل، مقسمة إلى OTP و ATM.
- **ملاحظات المشرف:** إمكانية كتابة ملاحظة سريعة وحفظها محلياً لكل عميل.
- **أزرار التحكم (Actions):** 
  - طلب كود OTP.
  - طلب صراف ATM.
  - إنهاء / رفض العملية.
  - قبول العملية وإظهار وثيقة التأمين.

## 5. الصفحات الجانبية (Views)
- **صفحة الإحصائيات (Stats):** 
  - ملخص الأرقام (الكل، متصل الآن، ناجح، مرفوض، بطاقات).
  - عرض جدول بيانات العملاء أو شبكة البطاقات بناءً على الفلتر.
- **صفحة البطاقات (Cards):** 
  - عرض شبكي (Grid) لجميع البطاقات المسحوبة.
  - زر لطباعة الصفحة (PDF Print).
- **صفحة المستخدمين (Users):**
  - نموذج لتغيير كلمة مرور الدخول يتطلب إدخال الرمز الرئيسي (Master Code).

---
**الخلاصة:**
لوحة التحكم الحالية غنية جداً بالميزات، وتعتمد بشكل كبير على التحديثات اللحظية (WebSockets). التصميم القادم يجب أن يحافظ على كل هذه الميزات، بل ويجعل الوصول إليها أسرع وأكثر وضوحاً عبر تجربة مستخدم (UX) محسنة وتصميم واجهة (UI) احترافي.
